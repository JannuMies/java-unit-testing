KISS

A couple of good programmers I know have desktop backgrounds that read "Keep it simple, stupid". It's a very good mantra, when writing code. My way of combining KISS and SRP in writing unit tests has been to mock every dependancy, thereby cutting down the cumulative multiplexity of different layers. Consider the following:

void updateTree(Tree treeToUpdate) { if(gardenValidator.isValid(treeToUpdate) { return garden.update(treeToUpdate); } else { throw new InvalidTreeException(treeToUpdate); } }

In this block, I am interested of only two things: A) If the tree is valid, it is passed to garden.update. B) If the tree is invalid, an exception is thrown. This means two tests. In this block the responsibility of validation is singled out, so I don't want to write different tests for all possible ways it can fail a validation, because that is the job for the tests testing GardenValidator.
