Doing TDD

I learned TDD by doing pair coding with patient people who showed me the ropes. If you don't have access to a mentor, a good way to learn is practice.

For instance, use 20 minutes on a single like these and repeat it every day of the week. Then switch to another for the next week. It's not important if you can't finish it in the 20 minutes, because you're going to do it again from the start the next day. The problem is only important if you are practicing algorithm solving, but since this should be an exercise in doing TDD, the only thing important is that you write it test-first.

If you are unsure how to begin, this will get you along.
